Examples

The examples, which are shown in this chapter of the documentation, provide a quick overview on how to use the library "AndroidMaterialViews". This project does contain the source code of an example app, which implements an use case of the library for demonstration purposes, as well.

Declaring a CircularProgressBar widget via XML

The XML code below shows how to declare a CircularProgressBar as a XML resource. The example contains all of the view's custom attributes.

<?xml version="1.0" encoding="utf-8"?>
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent" >

    <de.mrapp.android.view.CircularProgressBar
        android:id="@+id/circular_progress_bar"
        android:layout_width="@dimen/circular_progress_bar_size_normal"
        android:layout_height="@dimen/circular_progress_bar_size_normal"
        android:color="@color/circular_progress_bar_color"
        android:thickness="@dimen/circular_progress_bar_thickness_normal" />

</RelativeLayout>

All values, which are defined by using resource IDs in the example above, can also be defined by directly assigning the required value.

Instantiating a CircularProgressBar widget programmatically

The Java code below shows how a CircularProgressBar widget can be initialized programmatically. The attributes, which are configured by using the view's setter methods, are the same as shown in the previous example.

RelativeLayout parent = new RelativeLayout(this);
CircularProgressBar circularProgressBar = new CircularProgressBar(this);
circularProgressBar.setColor(getResourcces().getColor(R.color.circular_progress_bar_color));
circularProgressBar.setThickness(getResources().getDimensionPixelSize(
    R.dimen.circular_progress_bar_thickness_normal));
int size = getResources().getDimensionPixelSize(R.dimen.circular_progress_bar_size_normal);
parent.addView(circularProgressbar, size, size);

Declaring a FloatingActionButton widget via XML

The XML code below shows how a FloatingActionButton widget can be declared within a XML layout resource. The example contains all of the view's custom attributes.

<?xml version="1.0" encoding="utf-8"?>
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:custom="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    android:layout_height="match_parent" >

    <de.mrapp.android.view.FloatingActionButton
        android:id="@+id/floating_action_button"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:color="@color/floating_action_button_color"
        android:icon="@drawable/floating_action_button_icon"
        custom:size="normal"
        custom:activatedColor="@color/floating_action_button_activated_color"
        custom:pressedColor="@color/floating_action_button_pressed_color"
        custom:disabledColor="@color/floating_action_button_disabled_color"
        custom:visibilityAnimationDuration="500" />

</RelativeLayout>

Instantiating a FloatingActionButton widget programatically

The Java code below shows how a FloatingActionButton widget can be initialized programmatically. The attributes, which are configured by using the view's setter methods, are the same as shown in the previous example.

FloatingActionButton floatingActionButton = new FloatingActionButton(this);
floatingActionButton.setColor(getResources().getColor(R.color.floating_action_button_color));
floatingActionButton.setIcon(R.drawable.floating_action_button_icon);
floatingActionButton.setSize(Size.NORMAL);
floatingActionButton.setActivatedColor(getResources().getColor(
    R.color.floating_action_button_activated_color));
floatingActionButton.setPressedColor(getResources().getColor(
    R.color.floating_action_button_pressed_color));
floatingActionButton.setDisabledColor(getResources().getColor(
    R.color.floating_action_button_disabled_color));
floatingActionButton.setVisibilityAnimationDuration(500L);
